Process & store reference
MW.1 — Rate Limiter
Sliding-window in-memory limiter. 5 req/5 min on register, 10/5 min on login. Returns 429 on exceed.
MW.2 — Request ID
Reads X-Request-ID header or generates UUID v4. Stored on request.state and echoed in response header.
MW.3 — CORS
FastAPI CORSMiddleware. Currently allows all origins (allow_origins=["*"]). Should be restricted in production.
P.2 — JWT Decode
python-jose decodes HS256 JWT. Extracts sub (user_id), tenant_id, and ver (token_version) claims.
P.3 — Token Version Check
Compares JWT ver claim against User.token_version. Mismatch = revoked token → 401. Enables instant logout-all.
P.4 — Tenant Context
Resolves tenant_id from JWT, verifies active UserTenant membership, builds RequestContext(user, tenant_id, role).
P.5 — RBAC Check
require_permission() dependency. "Admin" role bypasses check. Others need matching RolePermission row. 403 on failure.
P.6 — Business Logic
Endpoint handlers in auth, tenants, roles, modules, audit routers. Reads/writes via SQLAlchemy ORM session.
P.7 — Audit Log Write
write_audit_log() appends action, resource, JSON details, IP, request_id, and user-agent to AuditLog table.
D1–D7 — PostgreSQL
Single Postgres 16 database. SQLAlchemy + psycopg2 driver. Pool with pre-ping enabled. Alembic manages migrations.
D8 — Rate Limit Store
SlidingWindowRateLimiter — Python dict of deques, thread-safe via threading.Lock(). Resets in-process only; not distributed.
P.8 — Response Serialise
Pydantic v2 response_model validation and JSON serialisation. x-request-id header injected by request_id_middleware.
