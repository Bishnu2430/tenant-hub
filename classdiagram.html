Model reference
User
Global identity. Holds credentials, token_version for revocation, links to many tenants via memberships.
Tenant
An organization instance. Industry-typed (school/hospital/hrms/ecommerce), localization-aware with country, currency, timezone.
UserTenant
Membership join table. Carries role assignment and enables tenant context-switching via JWT.
Role
Named permission bundle, scoped per tenant. "Admin" is treated as tenant superuser — bypasses RBAC checks.
Permission
Atomic capability, e.g. attendance:write. Global registry seeded from DEFAULT_PERMISSIONS.
RolePermission
Many-to-many join between Role and Permission. Checked at runtime by require_permission() dependency.
Module
Domain plugin: school, hospital, HRMS, e-commerce. Global registry; toggled per-tenant via TenantModule.
TenantModule
Activates a module for a specific tenant. Supports re-enabling (is_enabled flag) without duplicate rows.
Subscription
Plan tier (basic/pro/enterprise) with expires_at tracking. Old subscriptions deactivated on new plan creation.
FeatureToggle
Per-tenant key/bool flags enabling granular feature rollout without code deployment.
AuditLog
Append-only action record. Stores actor, resource, IP, request_id, user-agent, and JSON details payload.
